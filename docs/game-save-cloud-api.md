# Ray Game Cloud Save API Contract

Frontend support is implemented in `/arcade/js/cloud-save.js`. The static site remains fully playable without this API: all progress saves locally first, then cloud sync is attempted only for logged-in Ray accounts.

## Auth

Reuse the existing Ray member session cookie.

- `GET /auth/me` must return `{ ok: true, user: { id/email/role, email_verified } }` for signed-in users.
- Game save endpoints must return `401` or `403` for anonymous/unverified users.

## D1 schema

```sql
CREATE TABLE IF NOT EXISTS user_game_saves (
  user_id TEXT NOT NULL,
  game_id TEXT NOT NULL,
  slot TEXT NOT NULL DEFAULT 'default',
  revision INTEGER NOT NULL DEFAULT 1,
  updated_at INTEGER NOT NULL,
  device_id TEXT NOT NULL,
  checksum TEXT,
  payload_json TEXT NOT NULL,
  PRIMARY KEY (user_id, game_id, slot)
);

CREATE INDEX IF NOT EXISTS idx_user_game_saves_user_updated
  ON user_game_saves(user_id, updated_at DESC);
```

## Endpoints

### `GET /game-saves/:gameId`

Returns the current user's save for one game.

```json
{
  "ok": true,
  "save": {
    "gameId": "arcade",
    "slot": "default",
    "revision": 7,
    "updatedAt": 1782860000000,
    "deviceId": "ray-device-...",
    "checksum": "9f08aabc",
    "payload": {}
  }
}
```

If no save exists, return `{ "ok": true, "save": null }`.

### `PUT /game-saves/:gameId`

Normal foreground save. `POST /game-saves/:gameId` should behave the same way and accept a `text/plain` JSON body; the frontend uses that POST form for mobile `pagehide`/background keepalive flushes to avoid CORS preflight loss.

Request body is generated by `RayCloudSave`:

```json
{
  "schema": 1,
  "appId": "arcade",
  "deviceId": "ray-device-...",
  "revision": 8,
  "updatedAt": 1782860000000,
  "reason": "bucket:stats",
  "checksum": "9f08aabc",
  "payload": {}
}
```

Response:

```json
{
  "ok": true,
  "gameId": "arcade",
  "slot": "default",
  "revision": 8,
  "updatedAt": 1782860000000,
  "checksum": "9f08aabc"
}
```

## Validation rules

- Allow game IDs currently used by the frontend: `arcade`, `starfall`.
- Enforce payload size, e.g. max 64 KiB per save.
- Store JSON as text; never execute or template it.
- Use authenticated `user_id` from session, never trust a client-sent user ID.
- Revision can be monotonic per user/game, but the frontend also merges progress defensively.

## Frontend behavior

- Local save always happens first.
- Anonymous users see `💾 本机存档 · 登录后云同步`.
- Logged-in users sync with debounce and `pagehide` flush.
- If this API is absent (`404`), the frontend disables cloud attempts temporarily and keeps local saves.
